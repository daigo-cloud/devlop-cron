# モデル運用ルール（指示役と実装役の分離・必須遵守）

```
  指示役（オーケストレーション）                実装役（コードを書く）
  ┌────────────────────────────┐   Agent ツールで委譲   ┌──────────────────┐
  │ 1. Opus 4.8                │ ────────────────────▶ │ Sonnet 5（必須）   │
  │ 2. 4.8の挙動不審→ Opus 4.7 │                       │ .claude/agents/    │
  │                            │ ◀──────────────────── │ sonnet-implementer │
  └────────────────────────────┘   実装結果を検収        └──────────────────┘
```

- **指示役は Opus 4.8**。タスク分解・設計判断・レビュー・検収を担う。
  - Opus 4.8 の挙動がおかしい場合は **Opus 4.7** を利用する。
  - （Fable はトークン消費が大きすぎたため指示役から除外・2026-07-06 オーナー指示）
- **コードの実装（作成・編集・リファクタリング・テスト修正）は必ず Sonnet 5 で行う。**
  指示役は自分でコードを書かず、Agent ツールで `.claude/agents/sonnet-implementer.md`
  （`model: sonnet` 固定）に実装を委譲し、返ってきた変更を検収してからコミットする。
- 例外はない。1行の修正でも実装は Sonnet 5 に委譲すること。
- **新しいリポジトリを作成するときは、この節と `.claude/agents/sonnet-implementer.md` を
  必ず初期コミットに含めること**（AGENTS.md が無ければ新規作成し、CLAUDE.md から `@AGENTS.md` で参照する）。
